# 步骤 18 — SSRF 漏洞测试

> **测试日期:** 2026-07-14  
> **漏洞位置:** `POST /fetch-url`  
> **漏洞类型:** 服务端请求伪造（SSRF）

---

## 漏洞概述

`/fetch-url` 将用户输入的 URL 直接传给 `urllib.request.urlopen()`，无任何协议限制或目标过滤，攻击者可利用该功能读取本地文件、探测内网服务。

---

## 漏洞测试

| # | 攻击向量 | Payload | 结果 |
|:-:|----------|---------|:----:|
| 1 | 读取系统文件 | `file:///etc/passwd` | ✅ 读取到 root 用户 |
| 2 | 读取 hosts | `file:///etc/hosts` | ✅ 读取到 hosts 配置 |
| 3 | 读取项目源码 | `file:///app.py` | ✅ 读取到 Flask 代码 |
| 4 | 内网自身服务 | `http://127.0.0.1:5000/` | ✅ 可访问 |
| 5 | 内网端口探测 | `http://127.0.0.1:22/` | ✅ 可探测端口 |
| 6 | 内网 192.168 段 | `http://192.168.10.130:5000/` | ✅ 可访问 |
| 7 | 外网请求 | `http://example.com` | ✅ 正常请求 |

---

## 风险说明

| 风险 | 说明 |
|------|------|
| 文件读取 | 通过 `file://` 协议读取任意本地文件 |
| 源码泄露 | 读取 `app.py` 获取 `secret_key` 和数据库路径 |
| 内网探测 | 扫描内网存活主机和开放端口 |
| 跳板攻击 | 以当前服务器为跳板攻击内网或外网 |
